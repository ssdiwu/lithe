# 术语表

## Lithe

本项目独立维护的终端优先 AI 原生开发工作台。

_Avoid_：Terax 中文版、Terax CN。

## 工作区（Workspace）

当前打开的文件系统根目录，以及由此确定的文件、Git、shell 和 AI 工具授权
范围。

## 空间（Space）

一组可命名、可切换的工作状态，包含可选工作区根目录、运行环境、标签页和
活动标签页。

_Avoid_：项目；仅在确实指 Git 或构建项目时使用“项目”。

## 终端会话（Terminal session）

由一个 PTY 进程及其前端视图共同组成的可交互 shell 会话。

## Blocks 模式（Blocks mode）

把同一终端会话中的命令输入和输出按命令块组织的展示模式；它不是另一种
shell，也不是独立执行环境。

## 内置 AI 智能体（Built-in AI agent）

由 Lithe 自己的 provider、conversation、tool approval 和 session runtime
驱动的 AI 执行路径。

_Avoid_：Pi；Pi 属于终端智能体。

## 终端智能体（Terminal agent）

运行在 Lithe 终端会话中的外部 coding-agent CLI，例如 Pi、Claude Code、
Codex 或 Gemini CLI。

## 提供商（Provider）

向 Lithe 暴露一个或多个模型和调用端点的 AI 服务或本地推理运行时。

## 上游（Upstream）

作为选择性源码同步来源的 Terax Git 仓库；它不拥有 Lithe 的发布、更新、
数据或凭据。
